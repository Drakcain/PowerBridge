# QR / Import Contract

PowerBridge uses a local setup QR contract to transfer basic PC network setup data from PowerBridge Windows Companion to the PowerBridge Android app.

The QR contract is setup-only. It is not a wake command channel, relay command channel, authentication channel, or secret transfer mechanism.

## Current Schema

```text
powerbridge.local_setup.v1
```

This schema is generated by PowerBridge Windows Companion and accepted by Android import.

## Purpose

The contract prepares or updates an Android `Local Wi-Fi Wake` profile.

It allows Android to receive the local network values needed for Wake-on-LAN without requiring the user to manually type every field.

The Android app always previews the imported setup before saving it.

## Generated Fields

| Field          | Purpose                                                  |
| -------------- | -------------------------------------------------------- |
| `schema`       | Identifies the setup contract version                    |
| `pcName`       | Suggested profile or PC name                             |
| `targetMac`    | Target PC MAC address used for Wake-on-LAN               |
| `targetIp`     | Target PC IPv4 address at detection time                 |
| `gateway`      | Detected default gateway                                 |
| `subnetPrefix` | Detected subnet prefix length                            |
| `broadcastIp`  | Detected local broadcast IP                              |
| `adapterName`  | Detected Windows adapter label                           |
| `adapterType`  | User-facing adapter type, such as wired or wireless      |
| `source`       | Indicates the payload was generated by Windows Companion |
| `createdAt`    | Timestamp for when the payload was generated             |

## Forbidden Fields

The setup QR must never include credentials, relay secrets, account data, or private keys.

Forbidden fields include:

| Field           |
| --------------- |
| `username`      |
| `password`      |
| `adminPassword` |
| `relayToken`    |
| `token`         |
| `apiKey`        |
| `secret`        |
| `credential`    |
| `privateKey`    |
| `sshKey`        |
| `winrmPassword` |
| `rdpPassword`   |

Android import should reject payloads that include forbidden fields.

## Import Behavior

Android accepts the payload through:

```text
Settings -> PC Profiles -> Scan Setup QR
```

Expected Android behavior:

1. Scan the QR code.
2. Parse the `powerbridge.local_setup.v1` JSON.
3. Reject malformed payloads.
4. Reject payloads with forbidden fields.
5. Show an import preview.
6. Let the user choose `Create New Profile` or `Update Current Profile`.
7. Save the setup as a `Local Wi-Fi Wake` profile.
8. Keep relay tokens, passwords, credentials, and secrets out of the import path.

The imported profile should use:

```text
Local Wi-Fi Wake
```

The QR import flow should not silently activate a relay method.

## Windows Companion Behavior

PowerBridge Windows Companion generates the QR from the same JSON payload used by CLI output. There should be one import contract, not separate GUI-only, CLI-only, or QR-only schemas.

Expected Windows Companion behavior:

1. User selects a Windows adapter.
2. User clicks `Detect This PC`.
3. Companion detects local network values.
4. User clicks `Generate QR Code`.
5. Companion renders a scan-safe QR code from the `powerbridge.local_setup.v1` JSON.
6. Android scans and imports the payload.

The primary QR should remain scan-safe:

* black-on-white
* sufficient quiet zone
* readable by standard Android camera/scanner behavior
* no stylized color mode as the default scan target

## GUI Output Behavior

The Windows Companion GUI displays the QR on screen for scanning.

The simplified GUI pairing flow should keep the QR session-local unless an explicit export feature is intentionally added later.

Current expected GUI behavior:

* display QR for the current session
* do not require clipboard use
* do not require admin rights
* do not expose relay tokens
* do not expose credentials
* do not act as a wake command sender

CLI mode may generate JSON or QR output for testing or troubleshooting, but generated files may contain real local machine values and should be treated as machine-local setup data.

## Security Rules

The QR contract must remain secret-free.

The QR must not include:

* relay tokens
* passwords
* account credentials
* API keys
* private keys
* Windows login data
* remote desktop credentials
* WinRM credentials
* SSH credentials
* cloud service credentials

The QR should contain only local setup values needed to prepare a Wake-on-LAN profile.

## Public-Safety Rules

Generated setup payloads may contain real local network values, including MAC addresses and private IP addresses.

Do not publish generated QR payloads, generated QR screenshots, generated JSON files, or diagnostics containing real setup values unless intentionally redacted.

Public examples should use placeholders only.

Safe example values:

```json
{
  "schema": "powerbridge.local_setup.v1",
  "pcName": "Gaming PC",
  "targetMac": "AA:BB:CC:DD:EE:FF",
  "targetIp": "192.168.1.100",
  "gateway": "192.168.1.1",
  "subnetPrefix": 24,
  "broadcastIp": "192.168.1.255",
  "adapterName": "Ethernet",
  "adapterType": "wired",
  "source": "windows_companion",
  "createdAt": "2026-01-01T12:00:00Z"
}
```

## Non-Goals

The QR contract is not:

* a wake command
* a relay command
* a remote-control channel
* a credential exchange
* a cloud pairing token
* a Windows agent protocol
* a smart-home protocol
* a remote desktop setup flow

## Compatibility Rule

Future versions may add optional non-secret fields, but they must not break `powerbridge.local_setup.v1` import compatibility.

If a future schema is needed, it should be versioned explicitly instead of changing the meaning of existing fields silently.

Example future schema naming:

```text
powerbridge.local_setup.v2
```

Until then, the QR contents should stay plain `powerbridge.local_setup.v1` JSON.
